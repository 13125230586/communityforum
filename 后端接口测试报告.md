## 四 测试结论概览

### 4.1 当前状态定义

| 状态 | 说明 |
| --- | --- |
| 代码走查通过 | 已确认接口存在 且业务代码已实现 |

### 4.2 总体结论

- 接口分层清晰 前后台边界明确
- 登录态依赖 Session 需要浏览器或支持 Cookie 的测试工具保持会话
- 后台接口统一使用 `@AuthCheck` 做管理员权限校验
- 操作日志接口已具备自动落库能力 适合纳入后台联调验证
- 文件上传接口依赖真实 OSS 配置 未配置 OSS 时需单独验证异常分支
- 当前最适合的测试方式：先导入 SQL 再按本报告顺序进行 Postman 或 Apifox 联调

## 五 通用测试规则

### 5.1 返回结构

所有接口统一返回：

```json
{
  "code": 0,
  "data": {},
  "message": "ok"
}
```

### 5.2 重点校验码

| code | 含义 | 典型场景 |
| --- | --- | --- |
| `0` | 成功 | 正常请求 |
| `40000` | 请求参数错误 | 参数缺失 参数非法 |
| `40100` | 未登录 | 需要登录但未带 Session |
| `40101` | 无权限 | 普通用户访问后台接口 |
| `40400` | 请求数据不存在 | 传入不存在的帖子 用户 评论等 |
| `50001` | 操作失败 | 业务操作未成功 |
| `50000` | 系统内部异常 | 运行态异常 |

### 5.3 通用必测项

以下测试点适用于绝大多数接口：

- 必填参数缺失
- 参数格式非法
- 资源不存在
- 未登录访问
- 普通用户访问后台接口
- 分页边界值 `current=1` `pageSize=10` `pageSize=0` `pageSize=100`
- 重复提交
- 删除后再次查询
- 逻辑删除数据是否不再出现在普通列表中

## 八 参数异常专项测试

### 8.1 用户模块

- 注册 `userAccount` 少于 4 位
- 注册 `userPassword` 少于 8 位
- 注册两次密码不一致
- 登录账号为空或密码为空

### 8.2 帖子模块

- 发帖 `boardId` 为空
- 发帖 `postTitle` 为空
- 发帖 `contentType` 为空
- 发帖 `content` 为空
- 发帖携带不存在标签ID
- 发帖携带禁用标签
- 编辑不存在帖子

### 8.3 评论模块

- 评论 `postId` 为空
- 评论内容为空
- 回复不存在父评论
- 评论已关闭的帖子

### 8.4 举报模块

- 举报 `bizType` 非 `post` 或 `comment`
- 举报 `bizId` 不存在
- 举报 `reportType` 为空

### 8.5 文件模块

- 上传空文件
- 上传超过 10MB 图片
- 上传非 `jpg jpeg png webp gif`
- 删除不存在文件
- 删除已删除文件
